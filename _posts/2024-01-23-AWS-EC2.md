## 📌 들어가며

이번 글에서는 AWS의 핵심 서비스인 **EC2(Elastic Compute Cloud)**의 개념을 정리한다. 클라우드에서 **가상 서버**를 빌려 쓰는 서비스로, 인스턴스 생성 방법(콘솔·CLI)과 구성 요소(유형·스토리지·네트워크·보안 그룹)를 살펴본다.

> **EC2란?** AWS가 제공하는 **가상 서버 서비스**. 가상화 기술을 기반으로 필요할 때 서버를 만들고, 크기를 늘리거나 줄이며 유연하게 확장할 수 있다. 다양한 운영체제를 지원해 여러 환경의 애플리케이션을 실행한다.

---

## 1. EC2 & AMI

EC2 인스턴스를 만들려면 **AMI(Amazon Machine Image)**가 필요하다. AMI는 인스턴스의 **밑그림(OS + 소프트웨어 구성)**이다.

| 개념 | 설명 |
|------|------|
| **EC2 인스턴스** | 실제로 동작하는 가상 서버(컴퓨터) |
| **AMI** | 인스턴스를 시작하기 위한 이미지(OS·SW) |

> 💡 AWS 제공 AMI를 그대로 쓸 수도 있고, **내가 세팅한 인스턴스를 커스텀 AMI로 만들어** 동일한 환경을 복제할 수도 있다. 이 커스텀 AMI가 뒤에서 Auto Scaling의 복제 기준이 된다.

## 2. 인스턴스 생성 방법

### 📌 콘솔로 생성

```
① AWS Console 로그인
② EC2 대시보드 이동
③ 인스턴스 시작 → AMI 선택
④ 인스턴스 유형(크기) 선택
⑤ 스토리지 추가
⑥ 태그 추가
⑦ 보안 그룹(방화벽) 구성
⑧ 키페어 선택 → 시작
```

### 📌 CLI로 생성

콘솔 클릭 대신 명령 한 줄로도 만들 수 있다.

```bash
aws ec2 run-instances \
  --image-id ami-xxxxxxxx \
  --instance-type t2.micro \
  --key-name YourKeyPairName \
  --subnet-id subnet-xxxxxxxx
```

> 💡 CLI 방식은 **자동화·반복 생성**에 유리하다. 스크립트에 넣어두면 동일한 인스턴스를 언제든 재현할 수 있어, 인프라를 코드로 다루는 첫걸음이 된다.

---

## 3. EC2 구성 요소

| 구성 요소 | 설명 |
|------|------|
| **인스턴스 유형** | 서버의 크기·성능 (`t2.micro`=초소형/경제형, `m5.large`=범용) |
| **스토리지** | 인스턴스 스토어(휘발성) / **EBS(영속성)** |
| **네트워크** | VPC 내 통신·인터넷 통신(가상 네트워크 인터페이스) |
| **보안 그룹** | 인바운드/아웃바운드를 제어하는 **가상 방화벽** |

> ⚠️ **인스턴스 스토어 vs EBS** — 인스턴스 스토어는 인스턴스를 종료하면 **데이터가 사라지는** 임시 저장소(캐시·임시 파일용)다. 보존이 필요한 데이터는 반드시 **EBS 볼륨**에 두어야 한다.

## 📝 정리

```
EC2
├─ 개념   클라우드 가상 서버
├─ AMI    인스턴스의 밑그림(OS·SW)
├─ 생성   콘솔(클릭) / CLI(자동화)
└─ 구성   유형 · 스토리지(EBS) · 네트워크 · 보안그룹
```

| 개념 | 한 줄 정의 |
|------|------|
| **EC2** | 클라우드 가상 서버 |
| **AMI** | 인스턴스 생성용 이미지 |
| **보안 그룹** | 인스턴스의 가상 방화벽 |

EC2는 AWS 컴퓨팅의 기본 단위인 **가상 서버**다. 핵심은 **AMI로 인스턴스를 찍어내고, 유형·스토리지·보안 그룹으로 성격을 정하는 것**이며, CLI를 쓰면 이 과정을 자동화할 수 있다.
