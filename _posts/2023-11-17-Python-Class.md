## 📌 들어가며

이번 글에서는 OOP의 꽃인 **클래스(class)**를 다룬다. C와 Java에서 기본 지식을 쌓고 들어와 비교적 수월했지만, 여전히 쉬운 개념은 아니다.

> **클래스란?**
> 함수처럼 코드를 정리하되, 한 발 더 나아가 **관련된 함수와 변수를 하나로 묶는다.** 통계학·경제학·정치학을 '사회과학'이라는 큰 카테고리로 묶는 것과 같다.

---

## 1. 클래스 정의와 self

```python
class 사회과학:
    def 통계학(self):
        print("통계학")
    def 경제학(self):
        print("경제학")
    def 정치학(self):
        print("정치학")
```

> 💡 클래스 안 함수(메소드)는 첫 파라미터로 **`self`**를 갖는다. 객체화됐을 때 **그 객체 자신**이 들어가는 자리다. 호출할 때는 생략한다.

```python
변수 = 사회과학()   # 객체화
변수.경제학()
# >>> 경제학
```

![Desktop View](/assets/img/Programming-Language/Python/Class/1.png)

> 💡 `변수1`, `변수2`처럼 객체를 여러 개 만들면 **메모리에 서로 다른 공간**에 할당된다. 복잡해 보이지만, 대규모 프로젝트에서 다형화된 클래스를 쓰려면 필수다.

## 2. 반환 vs 출력 메소드

```python
class 수업시간:
    def 예제1(self):
        return "어려워요"      # 반환
    def 예제2(self, 값):
        print("입력:", 값)      # 출력
```

| 메소드 | 동작 |
|------|------|
| `예제1()` | 값을 **반환** → `print(예제1())`처럼 출력해야 보임 |
| `예제2(값)` | 내부에서 직접 **출력** |

## 3. 파라미터가 있는 메소드

```python
class 예의바른학생:
    def 첫인사(self, 이름):
        print("안녕하세요")
        print(f"내이름은 {이름}!")
    def 작별인사(self):
        print("안녕히가세요")

홍길동 = 예의바른학생()
홍길동.첫인사("신동희")   # self는 자동, 이름만 전달
홍길동.작별인사()
```

> 💡 두 가지 호출 방식이 있다.
> - `홍길동.첫인사("신동희")` — 객체로 호출 (self 자동)
> - `예의바른학생.첫인사(홍길동, "신동희")` — self에 객체를 직접 전달

## 4. self로 객체의 변수 만들기

```python
class 사람:
    def 이름입력(self, 이름):
        self.이름 = 이름        # 객체 자신의 변수 생성
    def 출력하기(self):
        print(self.이름)

첫째 = 사람()
첫째.이름입력("신동희")
print(첫째.이름)
# >>> 신동희
```

> 💡 `self.이름 = 이름`은 **그 객체(첫째)가 가진 변수**를 만든다. 그래서 `첫째.이름`에 "신동희"가 저장된다. 객체마다 독립적인 상태를 갖게 하는 핵심이다.

## 📝 정리

```
파이썬 클래스
├─ 정의    class 이름:  + 메소드(첫 파라미터 self)
├─ 객체화  변수 = 클래스()  → 메모리에 독립 공간
├─ self    객체 자신 (호출 시 자동 전달)
└─ self.변수  객체마다 독립적인 상태 저장
```

| 개념 | 한 줄 정의 |
|------|------|
| **클래스** | 관련 함수·변수를 묶는 틀 |
| **self** | 메소드가 가리키는 객체 자신 |
| **self.변수** | 객체 고유의 인스턴스 변수 |

파이썬 클래스의 특징은 **`self`**다. "메소드의 첫 인자는 객체 자신"이라는 규칙만 잡으면, C/Java의 클래스 지식이 그대로 이어진다.
