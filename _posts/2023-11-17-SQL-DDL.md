## 📌 들어가며

이번 글에서는 **DDL(Data Definition Language, 데이터 정의어)**을 정리한다. `CREATE`, `ALTER`, `TRUNCATE`, `DROP` 등 테이블 구조를 다루는 명령어들이다.

> ⚠️ **DDL은 따로 커밋하지 않아도 즉시 반영**되고 **롤백이 불가능**하다. 그래서 사용할 때 매우 조심해야 한다.

---

## 1. CREATE — 테이블 생성

```sql
CREATE TABLE PRODUCT(
    PRODUCT_ID   NUMBER,
    PRODUCT_NAME VARCHAR2(30),
    MANUDATE     DATE
);

INSERT INTO PRODUCT(PRODUCT_ID, PRODUCT_NAME, MANUDATE)
    VALUES(1, 'CELLPHONE', TO_DATE('YYMMDD'));
```

![Desktop View](/assets/img/Database/SQL/DDL/1.png)

**테이블·열 이름 규칙:**

| 규칙 | 설명 |
|------|------|
| 중복 불가 | 같은 테이블명·같은 열 이름 불가 |
| 예약어 금지 | `SELECT`, `INSERT`, `FROM` 등 사용 불가 |
| 문자로 시작 | 한글·특수문자 가능하나 권장 안 함 |
| 적당한 길이 | 너무 긴 이름 지양 |

---

## 2. ALTER — 테이블 구조 변경

열 단위로 추가·수정·삭제·이름 변경을 한다.

| 명령 | 역할 |
|------|------|
| `ADD (열 타입)` | 열 추가 |
| `MODIFY (열 타입)` | 열 타입 변경 |
| `DROP COLUMN 열` | 열 삭제 |
| `RENAME COLUMN A TO B` | 열 이름 변경 |

```sql
ALTER TABLE PRODUCT ADD (FACTORY VARCHAR2(10));               -- 추가
ALTER TABLE PRODUCT MODIFY (PRODUCT_NAME VARCHAR2(50));       -- 타입 변경
ALTER TABLE PRODUCT DROP COLUMN PRODUCT_NAME;                 -- 삭제
ALTER TABLE PRODUCT RENAME COLUMN PRODUCT_NAME TO PRODUCT_NAMEE; -- 이름 변경
```

![Desktop View](/assets/img/Database/SQL/DDL/2.png)
![Desktop View](/assets/img/Database/SQL/DDL/3.png)

> ⚠️ `MODIFY`로 타입을 바꿀 때 **기존보다 작은 형으로는 변환 불가**하다. `DROP COLUMN`은 롤백이 안 되니 특히 조심하자.

---

## 3. TRUNCATE vs DROP

```sql
TRUNCATE TABLE PRODUCT;   -- 틀은 남기고 '내용만' 삭제
DROP TABLE PRODUCT;       -- 테이블을 '아예' 삭제
```

| 명령 | 삭제 대상 |
|------|------|
| `TRUNCATE` | 데이터(내용)만, **구조는 유지** |
| `DROP` | 테이블 **자체** 삭제 |

---

## 📝 정리

```
DDL (데이터 정의어)
├─ CREATE    테이블 생성
├─ ALTER     ADD/MODIFY/DROP COLUMN/RENAME (구조 변경)
├─ TRUNCATE  내용만 삭제 (구조 유지)
└─ DROP      테이블 자체 삭제
⚠️ 즉시 반영 + 롤백 불가 → 신중히!
```

| 개념 | 한 줄 정의 |
|------|------|
| **CREATE** | 테이블 생성 |
| **ALTER** | 열 단위 구조 변경 |
| **TRUNCATE vs DROP** | 내용만 삭제 vs 테이블 삭제 |

DDL은 테이블의 "구조"를 다루는 강력한 명령어다. **롤백이 불가능**하다는 점이 DML과의 결정적 차이이니, 실행 전에 항상 한 번 더 확인하는 습관을 들이자.
