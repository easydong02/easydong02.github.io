## 📌 들어가며

JPA의 지연 로딩은 **프록시(Proxy)** 객체 위에서 동작한다. 프록시는 실제 엔티티를 대신하는 가짜 객체로, 실제로 사용될 때 초기화되어 진짜 엔티티에 접근한다.

> **`em.getReference()`**는 실제 엔티티가 아니라 **프록시 객체**를 반환한다. (`em.find()`는 실제 엔티티)

---

## 1. 프록시의 특징 — 한 번만 초기화

```java
Member findMember = em.getReference(Member.class, member.getId());

System.out.println(findMember.getClass());     // 프록시 객체
System.out.println(findMember.getId());        // 쿼리 안 나감 (id는 이미 앎)
System.out.println(findMember.getUsername());  // ★ 여기서 쿼리 발생! (실제 값 필요)
System.out.println(findMember.getClass());     // 여전히 프록시!
```

> 💡 프록시는 **처음 사용할 때 한 번만 초기화**된다. 중요한 건, **프록시가 실제 엔티티로 바뀌는 게 아니라** 실제 엔티티에 **접근할 수 있게 링크**되는 것이다. `getReference()`로 가져온 객체는 초기화 후에도 계속 **프록시**다.

---

## 2. ⚠️ 타입 비교는 instanceof

프록시 때문에 `==` 비교가 예상과 다르게 나올 수 있다.

```java
// 둘 다 find → 같은 실제 엔티티 클래스
em.find(...).getClass() == em.find(...).getClass()          // true

// find vs getReference → 실제 엔티티 vs 프록시
em.find(...).getClass() == em.getReference(...).getClass()  // false!
```

> ⚠️ 그래서 타입 비교는 **`==`가 아니라 `instanceof`**를 써야 한다.

```java
findMember  instanceof Member   // true (실제든 프록시든)
findMember2 instanceof Member   // true
```

---

## 3. 영속성 컨텍스트와 프록시

| 상황 | 결과 |
|------|------|
| 이미 영속성 컨텍스트에 **엔티티**가 있으면 | `getReference()`도 **실제 엔티티** 반환 |
| 이미 **프록시**가 있으면 | `find()`도 **프록시** 반환 |

> 💡 JPA는 **같은 영속성 컨텍스트에서 같은 식별자면 항상 같은 객체(==)**임을 보장한다. 그래서 이미 있는 것과 타입을 맞추려 한다. (find 먼저면 실제, getReference 먼저면 프록시로 통일)

---

## 4. ⚠️ 준영속 상태의 프록시 초기화

```java
Member findMember = em.getReference(Member.class, member.getId());
em.detach(findMember);              // 준영속 상태로
String username = findMember.getUsername();   // ★ 예외 발생!
```

```
org.hibernate.LazyInitializationException:
    could not initialize proxy - no Session
```

> ⚠️ 프록시 초기화는 **영속성 컨텍스트의 관리**가 필요하다. `detach`/`clear`/트랜잭션 종료 등으로 **준영속 상태가 되면 초기화 시 `LazyInitializationException`**이 발생한다.

---

## 5. 프록시 상태 확인·강제 초기화

```java
// 초기화 여부 확인
emf.getPersistenceUnitUtil().isLoaded(findMember);   // false
findMember.getUsername();                            // 초기화 발생
emf.getPersistenceUnitUtil().isLoaded(findMember);   // true

// 강제 초기화
Hibernate.initialize(findMember);
```

## 📝 정리

```
JPA 프록시
├─ 특징   한 번만 초기화, getReference()가 반환
├─ 본질   실제 엔티티로 바뀌지 않고 '접근'만 가능
├─ 비교   == 대신 instanceof 사용
├─ 컨텍스트  같은 식별자면 같은 객체로 통일
└─ 주의   준영속 상태 초기화 → LazyInitializationException
```

| 개념 | 한 줄 정의 |
|------|------|
| **프록시** | 실제 엔티티를 대신하는 가짜 객체 |
| **getReference** | 프록시를 반환 |
| **instanceof** | 프록시/실제 무관하게 타입 비교 |
| **LazyInitializationException** | 준영속 프록시 초기화 실패 |

프록시는 지연 로딩의 핵심 메커니즘이다. **"프록시는 실제 엔티티로 바뀌지 않는다"**는 점, 그래서 **타입 비교는 `instanceof`**로 해야 한다는 점, 그리고 **준영속 상태에서 초기화하면 예외**가 난다는 점만 잡으면 지연 로딩을 안전하게 다룰 수 있다.
