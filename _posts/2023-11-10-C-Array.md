## 📌 들어가며

C를 배우다 보면 "같은 타입의 값 여러 개를 한꺼번에 다뤄야 하는" 순간이 온다. 학생 5명의 점수, 센서 100개의 측정값처럼. 이걸 변수 하나에 하나씩 담으면 `score1`, `score2`, … `score100`이 되어버린다. 이 문제를 해결하는 도구가 바로 **배열(array)**이다.

> **배열(array)이란?**
> 동일한 타입의 데이터 여러 개를 **하나의 이름**으로 묶어 관리하는 집합 데이터 구조.

이번 글에서는 배열의 **선언·초기화**부터 **크기 계산**, **문자열과의 관계**, **다차원 배열**까지 순서대로 정리한다.

---

## 1. 배열의 선언

배열 변수는 다음 두 가지 형식으로 선언한다.

```c
타입 배열변수명[배열길이];              // 길이만 지정
타입 배열변수명[] = { 초기값, ... };    // 초기값 개수로 길이 자동 결정
```

여기서 반드시 기억할 규칙 하나. **배열 첨자(index)는 0부터 시작한다.** 길이가 5인 배열이라면 유효한 인덱스는 `0 ~ 4`다.

```
int arr[5];

인덱스:   0        1        2        3        4
        ┌────────┬────────┬────────┬────────┬────────┐
        │ arr[0] │ arr[1] │ arr[2] │ arr[3] │ arr[4] │
        └────────┴────────┴────────┴────────┴────────┘
          ↑ 첫 번째                          ↑ 마지막 (길이-1)
```

실제로 배열을 하나 만들어 보자.

![Desktop View](/assets/img/Programming-Language/C/Array/1.png){: width="100%" }

`arr`이라는 이름의 `int` 타입 배열을 원소(방) 5개짜리로 선언했다. 인덱스 0부터 4까지 각각 1~5의 값을 넣었다. 이제 `printf("%d", arr[2]);`로 3번째 원소(`arr[2]`)의 값이 제대로 들어갔는지 확인한다.

![Desktop View](/assets/img/Programming-Language/C/Array/2.png){: width="100%" }

`arr[2]`, 즉 세 번째 원소의 값이 정확히 출력됐다.

## 2. 배열 초기화

배열을 선언만 하고 초기화하지 않으면 **쓰레기값(garbage value)**이 들어간다. 따라서 초기화는 중요하다. 초기화 방법은 크게 네 가지다.

| 방법 | 예시 | 설명 |
|------|------|------|
| ① 하나씩 대입 | `arr[0]=1; arr[1]=2; …` | 원소마다 직접 대입. 길이가 크면 번거로움 |
| ② 선언과 동시에 | `int arr[3] = {0,1,2};` | 가장 일반적인 방식 |
| ③ 첫 원소만 지정 | `int arr[100] = {0};` | 나머지 99개는 **자동으로 0** |
| ④ 길이 생략 | `int arr[] = {1,2,3};` | 초기값 개수로 길이(=3) 자동 결정 |

> 💡 **③번의 포인트**: `{0}`으로 첫 원소만 지정해도 **나머지 원소는 전부 0으로 초기화**된다. 이건 첫 값이 하필 0이라서가 아니라, 초기화 목록에 포함되지 않은 원소는 무조건 0으로 채워지기 때문이다. `int arr[100] = {7}` 이라 해도 나머지 99개는 7이 아니라 0이 된다.

## 3. 배열의 크기(size)와 길이(length)

두 개념은 이름이 비슷해서 헷갈리기 쉽지만 엄연히 다르다.

| 용어 | 의미 | 예시 (`int arr[3]`) |
|------|------|------|
| **크기(size)** | 배열이 차지하는 총 바이트 | `4 byte × 3 = 12 byte` |
| **길이(length)** | 원소의 개수 | `3` |

크기는 원소 타입의 크기에 따라 결정된다. `int`는 4byte이므로, `int` 타입에 길이 4인 배열은 총 `4 × 4 = 16 byte`가 된다.

여기서 `sizeof()` 연산자가 등장한다. **데이터의 크기를 바이트(정수)로 돌려주는** 연산자다.

```c
sizeof(int)     // 4
sizeof(char)    // 1
sizeof(double)  // 8
```

이걸 활용하면 배열의 길이를 직접 계산할 수 있다.

```c
배열 길이 = sizeof(배열이름) / sizeof(배열원소타입);
```

`sizeof(배열이름)`은 배열 **전체 크기**(원소타입 크기 × 길이)를 돌려주므로, 여기에 원소 하나의 크기로 나누면 길이만 남는다.

```
int arr[3];

sizeof(arr) = 4 × 3 = 12    (전체 크기)
sizeof(int) = 4             (원소 하나의 크기)
────────────────────────────────
길이 = 12 / 4 = 3  ✅
```

> ⚠️ C에서는 **길이 0짜리 배열은 선언할 수 없다.**

## 4. 배열에서 최대값·최소값 구하기

배열 하나를 만들어 최대값을 구해 보자.

![Desktop View](/assets/img/Programming-Language/C/Array/3.png){: width="100%" }

상수 `MAX`를 5로 지정하고 `for`문으로 길이 5인 `int` 배열 `arr`에 값 5개를 입력했다. 그렇다면 이 안에서 최대값은 어떻게 찾을까?

원소가 5개면 첫 번째와 두 번째를 비교하고, 그다음, 그다음… 결국 **4번의 비교**가 필요하다. 원소가 100개면 99번을 비교해야 하니 손으로는 불가능하다. 그래서 반복문을 쓴다.

![Desktop View](/assets/img/Programming-Language/C/Array/4.png){: width="100%" }

**동작 원리**는 이렇다.

```
1. max ← arr[0]              (첫 원소를 후보로 넣는다)
2. i = 1부터 마지막까지 반복
     if (arr[i] > max)       (더 큰 값을 만나면)
         max ← arr[i]        (후보를 갱신한다)
3. 반복이 끝나면 max = 최대값
```

`max`라는 변수에 첫 번째 원소를 넣고, `for`문으로 (i는 1부터) 더 큰 값을 만날 때마다 `max`를 갱신한다. 마지막 원소까지 돌면 `max`에 최대값이 남는다.

![Desktop View](/assets/img/Programming-Language/C/Array/5.png){: width="100%" }

> 💡 **최소값**은 부등호 방향만 반대(`arr[i] < min`)로 바꾸면 그대로 구할 수 있다.

## 5. 배열과 문자열(string)

C에는 별도의 문자열 타입이 없다. C에서 **"문자열"의 정체는 `char[]` 배열**이다. 정확히는 **끝이 `'\0'`(null 문자)로 끝나는 `char[]` 배열**이다.

> **null 문자(`'\0'`)는 공백이 아니다.**
> 공백(`' '`)은 눈에 보이지 않을 뿐 엄연한 하나의 문자지만, null은 **0byte 크기의, 문자열의 끝을 알리는 표식**이다.

예를 들어 `char str1[20] = "nice";`는 각 원소에 `n, i, c, e`가 들어가고, 5번째 원소 `str1[4]`에는 자동으로 null 문자가 들어간다.

```
char str1[20] = "nice";

인덱스:  0    1    2    3    4     5 ...
       ┌────┬────┬────┬────┬──────┬─────
       │ n  │ i  │ c  │ e  │ \0   │ ...
       └────┴────┴────┴────┴──────┴─────
                            ↑ 여기서 문자열 끝
```

그런데 이렇게 하면 어떻게 될까?

```c
char str2[] = {'n','i','c','e'};   // null 문자가 없다!
```

![Desktop View](/assets/img/Programming-Language/C/Array/6.png){: width="100%" }

`nice` 뒤에 이상한 문자가 딸려 나온다. 컴파일러는 **null 문자를 만날 때까지** 계속 읽는데, 위 배열에는 끝에 null이 없어서 메모리의 엉뚱한 값까지 문자열로 읽어버린 것이다. 따라서 아래처럼 **null 문자를 직접 넣어줘야** 한다.

```c
char str2[] = {'n','i','c','e','\0'};   // 이제 nice만 출력된다
```

## 6. 다차원 배열(n-dimensional array)

**다차원 배열**은 배열 첨자를 여러 개 사용한 것이다. 첨자 2개를 쓰면 **2차원 배열**이다.

**어디에 쓸까?** 필드가 2개 이상인 자료를 다룰 때 유용하다. 예를 들어 학생 5명이 각각 4과목 시험을 봤고 그 점수를 저장할 때, `arr[학생][과목]` 형태로 다루면 깔끔하다.

**선언**은 이렇게 한다.

```c
int arr[4][3];   // 4행 3열 (행렬로 보면 4×3)
```

**초기화**는 1차원 배열과 원리가 같지만, `for`문을 **중첩**해서 사용한다.

![Desktop View](/assets/img/Programming-Language/C/Array/7.png){: width="100%" }

바깥 `for`가 행을, 안쪽 `for`가 열을 돌면서 `[0][0], [0][1], [0][2], [1][0], …` 순서로 값을 채운다.

```
        열0   열1   열2
행0   [0][0][0][1][0][2]
행1   [1][0][1][1][1][2]
행2   [2][0][2][1][2][2]
행3   [3][0][3][1][3][2]
```

첫 원소만 지정하고 나머지를 0으로 초기화하는 방법(`int arr[4][3] = {0};`)도 1차원 배열과 동일하게 동작한다.

---

## 📝 정리

```
배열(array)
├─ 선언       int arr[5];  또는  int arr[] = {...};  (인덱스는 0부터)
├─ 초기화     ① 하나씩  ② 선언과 동시에  ③ {0}으로 전체 0  ④ 길이 생략
├─ 크기/길이  size = 총 byte,  length = 원소 개수
│             길이 = sizeof(배열) / sizeof(원소타입)
├─ 최대/최소  첫 원소를 후보로 두고 반복하며 갱신
├─ 문자열     char[] 배열 + 끝에 반드시 '\0'
└─ 다차원     arr[행][열], 초기화는 중첩 for문
```

| 개념 | 한 줄 정의 |
|------|------|
| **배열** | 같은 타입 데이터를 하나의 이름으로 묶은 집합 |
| **첨자(index)** | 원소의 위치. **0부터 시작** |
| **쓰레기값** | 초기화하지 않은 원소에 들어있는 의미 없는 값 |
| **sizeof** | 데이터의 크기를 byte로 돌려주는 연산자 |
| **null 문자** | `'\0'`, C 문자열의 끝을 알리는 표식 |
| **다차원 배열** | 첨자를 2개 이상 사용한 배열 |

오늘은 배열에 대해서 알아봤다. 인덱스가 0부터 시작한다는 것, 그리고 C 문자열은 반드시 `'\0'`으로 끝나야 한다는 두 가지만 확실히 잡고 가자!
