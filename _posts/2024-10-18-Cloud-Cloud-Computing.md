## 📌 들어가며

이번 글에서는 **클라우드 컴퓨팅**의 개념과 이점, 그리고 **AWS 글로벌 인프라**(리전·가용 영역)를 정리한다. 클라우드 인프라 구축 프로젝트의 첫 단추로, 왜 물리 서버 대신 클라우드를 쓰는지부터 짚는다.

> **클라우드 컴퓨팅이란?** 컴퓨팅 파워·데이터베이스·스토리지 같은 IT 리소스를 **인터넷을 통해 온디맨드(on-demand)로 제공**하고, **사용한 만큼만 비용을 지불**하는 방식. 물리 데이터센터·서버를 직접 사고 유지하는 대신, AWS 같은 **클라우드 공급자(CSP)**로부터 필요할 때 빌려 쓴다.

![클라우드 컴퓨팅](/assets/img/Cloud/cloud-computing/image.png)

## 1. IT 서비스 모델 & 배포 모델

클라우드는 **관리 책임의 범위(서비스 모델)**와 **인프라 위치(배포 모델)**로 나눠 이해할 수 있다.

| 서비스 모델 | 제공 범위 | 예 |
|------|------|------|
| **IaaS** | 인프라(서버·네트워크) | EC2 |
| **PaaS** | 실행 플랫폼 | Elastic Beanstalk |
| **SaaS** | 완성된 소프트웨어 | Gmail |

| 배포 모델 | 설명 |
|------|------|
| **퍼블릭 클라우드** | CSP의 공용 인프라 |
| **프라이빗 클라우드** | 전용 인프라 |
| **하이브리드** | 퍼블릭 + 온프레미스 결합 |

## 2. 클라우드 컴퓨팅의 6가지 이점

| 이점 | 설명 |
|------|------|
| **자본비용 → 가변비용** | 초기 대규모 투자(CapEx) 대신 쓴 만큼(OpEx) 지불 |
| **규모의 경제** | 전체 사용량을 모아 더 낮은 단가 실현 |
| **용량 추정 불필요** | 미리 서버 규모를 예측할 필요 없음 |
| **속도·민첩성 향상** | 몇 분 만에 리소스 확보 |
| **운영/유지관리 비용 절감** | 데이터센터 관리 부담 제거 |
| **몇 분 만에 전 세계 배포** | 글로벌 리전으로 즉시 확장 |

> 💡 클라우드의 본질은 **"소유에서 사용으로"**의 전환이다. 서버를 직접 사서 운영하면 미래 트래픽을 예측해 미리 투자해야 하지만, 클라우드는 필요할 때 늘리고 안 쓰면 줄여 **낭비 없이 비용을 트래픽에 맞춘다.**

## 3. AWS 글로벌 인프라 — 리전과 가용 영역

AWS 인프라는 **리전(Region) → 가용 영역(AZ) → 데이터센터**의 계층으로 구성된다.

![글로벌 인프라](/assets/img/Cloud/cloud-computing/image%209.png)

| 단위 | 설명 |
|------|------|
| **리전(Region)** | 지리적 영역(전 세계 약 34개). 서로 완전히 분리·이중화 |
| **가용 영역(AZ)** | 리전 내 물리적으로 분리된 데이터센터 그룹(약 108개) |
| **데이터센터** | AZ를 구성하는 실제 물리 시설 |

- **리전**: 리전 간 통신은 AWS 백본 네트워크를 사용하고, 데이터 복제 여부는 **고객이 제어**한다.
- **가용 영역**: 물리적으로 분리되어 **내결함성**을 제공한다. 지진 지역(예: 일본)에서는 각 AZ가 서로 다른 전력망·단층선에 배치된다. AZ 간은 **고속 프라이빗 네트워크**로 연결되어 동기 복제가 가능하다.

![리전](/assets/img/Cloud/cloud-computing/image%2010.png)

![리전 선택 고려사항](/assets/img/Cloud/cloud-computing/image%2011.png)

![가용 영역](/assets/img/Cloud/cloud-computing/image%2012.png)

> ⚠️ **여러 가용 영역에 걸쳐 리소스를 복제**하는 것이 권장된다. 한 AZ에 장애가 나도 다른 AZ가 이어받아 서비스가 중단되지 않는 **고가용성**을 확보할 수 있기 때문이다.

## 📝 정리

```
클라우드 컴퓨팅
├─ 정의   IT 리소스를 인터넷으로 온디맨드·종량제 제공
├─ 모델   서비스(IaaS/PaaS/SaaS) + 배포(퍼블릭/프라이빗/하이브리드)
├─ 이점   CapEx→OpEx, 규모의 경제, 민첩성, 글로벌 배포
└─ 인프라 리전 > 가용 영역(AZ) > 데이터센터
```

| 개념 | 한 줄 정의 |
|------|------|
| **클라우드** | 필요할 때 빌려 쓰는 IT 리소스 |
| **리전** | 지리적으로 분리된 영역 |
| **가용 영역(AZ)** | 리전 내 물리적으로 분리된 DC |

클라우드의 핵심은 **소유 대신 사용**, 그리고 **여러 AZ 분산으로 고가용성 확보**다. 리전·가용 영역 개념은 앞으로의 모든 인프라 설계(VPC·서브넷·이중화)의 토대가 된다.
