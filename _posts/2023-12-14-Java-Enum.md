## 📌 들어가며

상태 코드나 분류처럼 **정해진 값의 집합**을 다룰 때, 예전에는 `final` 상수를 나열해서 썼다. 하지만 여러 클래스에서 쓰기 번거롭고, 관련된 값들을 한데 묶어 관리하기 어려웠다. 이 문제를 해결하는 것이 **Enum**이다.

> **Enum이란?**
> "Enumeration(열거)"의 약자. **로직보다는 데이터를 열거한 요소들의 집합**이다. 자바에서 상수 처리를 도맡는다.

---

## 1. 과거의 상수 처리

예전에는 `final` 제어자로 상수를 만들었다.

```java
class PastFinal {
    private final static int ONE = 1;

    public static void main(String[] args) {
        System.out.println(PastFinal.ONE);
    }
}
```

> ⚠️ 이 방식은 여러 클래스에서 쓰기 번거롭고, 관련 상수들을 **캡슐화**할 수 없다. "이 상수들을 하나로 묶을 수 없을까?"에서 나온 것이 **Enum**이다.

---

## 2. Enum 선언

각 상수에 **코드와 설명을 함께 매핑**하는 Enum을 만들어 보자.

```java
// OutputType.java
public enum OutputType {

    INSPEC ("01", "점검 중"),
    SUCCESS("02", "정상 종료"),
    FAIL   ("03", "처리 오류"),
    SB     ("04", "반송"),
    HW     ("05", "수작업(재할당)"),
    UNKNOWN("",   "unknown");

    OutputType(String code, String description) {   // 생성자
        this.code = code;
        this.descriptions = description;
    }

    private final String code;
    private final String descriptions;

    public String getCode()         { return code; }
    public String getDescriptions() { return descriptions; }

    // 코드로 Enum 객체를 찾는 메소드
    public static OutputType getByCode(String code) {
        for (OutputType type : OutputType.values()) {
            if (type.getCode().equalsIgnoreCase(code)) {
                return type;
            }
        }
        return OutputType.UNKNOWN;
    }
}
```

구조를 하나씩 뜯어보면 이렇다.

| 구성 요소 | 역할 |
|------|------|
| `INSPEC("01","점검 중")` | 각 Enum 객체 선언 (생성자 인자 = 코드, 설명) |
| `private final String code, descriptions` | 각 객체가 가질 필드 |
| 생성자 `OutputType(...)` | 코드·설명을 필드에 매핑 |
| `getCode()` / `getDescriptions()` | 매핑된 값을 꺼내는 Getter |
| `getByCode(code)` | 코드로 해당 Enum 객체를 찾아 반환 |

> 💡 `values()`는 Enum에 기본 제공되는 메소드로, **그 Enum의 모든 객체**를 배열로 돌려준다. `getByCode()`는 이걸 순회하며 `getCode()`로 비교해 하나의 객체를 반환한다.

---

## 3. Enum 사용

사용은 아주 쉽다.

```java
// 코드 출력
System.out.println(OutputType.INSPEC.getCode());
// → 01

// 설명 출력
System.out.println(OutputType.INSPEC.getDescriptions());
// → 점검 중

// 코드로 객체를 찾아 설명 출력
OutputType findType = OutputType.getByCode("03");
System.out.println(findType.getDescriptions());
// → 처리 오류
```

| 요청 | 코드 | 결과 |
|------|------|------|
| INSPEC의 코드 | `OutputType.INSPEC.getCode()` | `01` |
| INSPEC의 설명 | `OutputType.INSPEC.getDescriptions()` | `점검 중` |
| 코드 03의 설명 | `getByCode("03").getDescriptions()` | `처리 오류` |

## 📝 정리

```
Enum
├─ 개념   정해진 값의 집합을 캡슐화한 상수 타입
├─ 선언   객체마다 코드·설명 등 필드를 생성자로 매핑
├─ 기본   values()로 전체 객체 순회 가능
└─ 활용   getByCode() 같은 정적 메소드로 코드↔객체 변환
```

| 개념 | 한 줄 정의 |
|------|------|
| **Enum** | 관련 상수를 필드·메소드와 함께 묶은 타입 |
| **values()** | Enum의 모든 객체를 반환하는 기본 메소드 |
| **getByCode 패턴** | 외부 코드값을 Enum 객체로 매핑 |

Enum은 단순한 상수 나열을 넘어, **값과 그에 딸린 데이터·동작을 하나로 캡슐화**한다. 상태 코드 관리처럼 "정해진 값의 집합"을 다룰 때 `final` 상수보다 훨씬 안전하고 표현력이 좋다.
