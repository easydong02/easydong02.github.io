## 📌 들어가며

이번 글에서는 자바 네트워크 프레임워크 **Netty**를 정리한다.

![Desktop View](/assets/img/Network/Netty/Netty/1.jpg)

> **Netty란?** 자바 환경에서 네트워크 프로그래밍을 손쉽게 해주는 프레임워크. **고성능·고안정성** 프로그램을 만들 수 있다. 순수 소켓 프로그래밍보다 **추상화**로 훨씬 간결하다.

```
Client ─────────────────────────────────── Server
 Outbound  ──►  Send (몇시에요?)  ──►        Inbound
 Inbound   ◄── (오후 8시) Response ◄──       Outbound
```

> **Netty = 비동기(Async) + 논블로킹(Non-blocking) + 이벤트 기반(Event-driven)**

---

## 1. 동기/비동기 vs 블로킹/논블로킹

흔히 혼동하지만 **서로 별개의 개념**이다.

| 개념 | 기준 | 정의 |
|------|------|------|
| **동기/비동기** | 작업의 **시작·종료 시간** | 동기: 시간을 맞춤 / 비동기: 각자 별도 |
| **블로킹/논블로킹** | 다른 주체를 **기다리는가** | 블로킹: 기다림 / 논블로킹: 계속 진행 |

![Desktop View](/assets/img/Network/Netty/Netty/2.png)

> 💡 **블로킹 예**: JDBC로 DB에 질의하고 결과를 받을 때까지 대기. **논블로킹 예**: 요청만 하고 결과를 기다리지 않고 자기 일을 계속.

**비동기 + 논블로킹 조합**은 자원이 충분하면 효율이 좋다. 자기 작업이 멈추지 않고, 결과가 나오면 **콜백**으로 처리한다.

| 적합한 상황 | 예 |
|------|------|
| 대규모·동기 불필요 | 대량 푸시 메시지 전송 |
| 여러 외부 API 동시 호출 | API 병렬 호출 |

---

## 2. 이벤트 기반 프로그래밍

> **이벤트를 미리 정의**하고, 이벤트 발생 시 실행될 코드를 준비한다. (논블로킹 소켓의 **Selector** I/O 감지가 대표적)

**네트워크에서 이벤트 발생 주체는 '소켓'**이며, 이벤트 종류는 **소켓 연결·데이터 송수신**이다.

![Desktop View](/assets/img/Network/Netty/Netty/5.png)

> 💡 네티는 채널에 직접 읽고 쓰지 않고 **데이터 핸들러**를 통해 작업한다. 그 장점은:
> - 서버 코드를 **클라이언트에서도 재사용**
> - **이벤트별 로직 분리** → 깔끔
> - **에러 이벤트도 함께 정의** → 에러 처리 부담↓

---

## 3. ChannelPipeline — 네티의 핵심

> 각 TCP 연결(`SocketChannel`)은 **ChannelPipeline**을 가지며, 이는 **ChannelHandler 인스턴스의 리스트**다.

![Desktop View](/assets/img/Network/Netty/Netty/6.png)

```
소켓 → [ChannelInboundHandler 체인] → 애플리케이션   (수신)
애플리케이션 → [ChannelOutboundHandler 체인] → 소켓   (송신)
```

| 방향 | 핸들러 체인 |
|------|------|
| **수신(Inbound)** | 소켓 데이터 → Inbound 체인 → 앱 |
| **송신(Outbound)** | 앱 데이터 → Outbound 체인 → 소켓 |

> 💡 이 체인 덕분에 **네트워크 통신 시 적절한 처리를 미리** 넣을 수 있다. 예를 들어 객체 전송 시, 체인 안에서 **직렬화 → 압축 → 바이트 전송**이 자동으로 처리된다. 한 번 구현해두면, 개발자는 그냥 객체를 **`write()`**만 하면 전송된다.

## 📝 정리

```
Netty
├─ 정체     자바 네트워크 프레임워크 (고성능·간결)
├─ 특징     비동기 + 논블로킹 + 이벤트 기반
├─ 개념     동기/비동기(시간) vs 블로킹/논블로킹(대기)
├─ 이벤트   소켓 이벤트(연결·송수신)를 핸들러로
└─ 핵심     ChannelPipeline = 핸들러 체인 (Inbound/Outbound)
```

| 개념 | 한 줄 정의 |
|------|------|
| **Netty** | 자바 고성능 네트워크 프레임워크 |
| **비동기/논블로킹** | 기다리지 않고 콜백으로 |
| **ChannelPipeline** | 데이터 처리 핸들러 체인 |

Netty의 핵심은 **ChannelPipeline(핸들러 체인)**으로 네트워크 처리를 추상화하는 것이다. 직렬화·압축 같은 반복 처리를 체인에 넣어두면, 개발자는 `write()` 한 번으로 복잡한 네트워크 통신을 간결하게 다룰 수 있다.
