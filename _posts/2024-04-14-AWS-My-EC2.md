## 📌 들어가며

이번 글에서는 AWS의 기본 개념(클라우드·리전·가용 영역)을 정리하고, 가장 핵심 서비스인 **EC2(가상 서버)**를 직접 생성·접속한 뒤, 클라우드 IDE인 **Cloud9**까지 만들어 본다.

> **AWS란?** 전 세계에서 쓰이는 대표적인 **클라우드 플랫폼**. 스타트업부터 대기업·정부 기관까지, 필요한 컴퓨팅 자원을 필요할 때 네트워크로 빌려 쓰는 **종량 과금제** 인프라를 제공한다.

---

## 1. 클라우드 & AWS의 장점

퍼블릭 클라우드는 *"관리 노력을 최소화하면서 필요할 때 즉시 빌려 쓰는 컴퓨팅 자원의 공유 풀"*이다. 온프레미스(자체 서버)와 비교하면 차이가 뚜렷하다.

| 항목 | **온프레미스** | **AWS(클라우드)** |
|------|----------------|-------------------|
| 초기 비용 | 서버 구매 필요 | **없음(종량제)** |
| 구축 속도 | 느림 | **빠름** |
| 리소스 확장 | 사전 확보 필요 | **필요 시 즉시** |
| 고가용성 | 직접 구성 | **서비스로 제공** |
| 자동화 | 제한적 | **API로 용이** |

---

## 2. 리전(Region)과 가용 영역(AZ)

AWS 인프라는 **리전 → 가용 영역**의 계층으로 나뉜다.

```
ap-northeast-2 (서울 리전)
├─ ap-northeast-2a  (가용 영역 AZ)
├─ ap-northeast-2b  (가용 영역 AZ)
├─ ap-northeast-2c  (가용 영역 AZ)
└─ ap-northeast-2d  (가용 영역 AZ)
```

| 단위 | 설명 |
|------|------|
| **리전(Region)** | 가장 큰 단위(지리적 위치) |
| **가용 영역(AZ)** | 리전 내 물리적으로 분리된 데이터센터 |

> 💡 여러 AZ에 자원을 분산하면, 한 AZ에 장애가 나도 다른 AZ가 서비스를 이어받아 **고가용성**을 확보할 수 있다. 이것이 AWS 아키텍처 설계의 기본 원칙이다.

## 3. EC2 생성

> **EC2(Elastic Compute Cloud)란?** 클라우드에서 크기 조정이 가능한 **가상 서버(컴퓨터)**를 제공하는 서비스. 웹 콘솔에서 몇 번의 클릭으로 서버를 띄우고 제어할 수 있다.

**AMI**(가상 머신 이미지)로 **Amazon Linux 2**를 선택하고, 프리 티어 인스턴스 유형을 고른다. 키페어는 **RSA + .pem**으로 새로 만든다.

![Untitled](/assets/img/Infra/AWS/ec2/2.png)

![Untitled](/assets/img/Infra/AWS/ec2/3.png)

보안 그룹에는 **HTTP(위치 무관)**과 **모든 ICMP-IPv4(위치 무관)**를 추가하고 인스턴스를 시작한다.

![Untitled](/assets/img/Infra/AWS/ec2/4.png)

> ⚠️ **키페어(.pem) 파일은 최초 생성 시에만 다운로드**된다. 잃어버리면 재발급이 안 되므로 잘 보관해야 한다. ICMP를 열어야 `ping` 테스트가 가능하다.

---

## 4. EC2 접속 & 웹 서버 설치

키페어 파일이 있는 위치에서 cmd(터미널)를 열고 SSH로 접속한다.

```bash
ssh -i <키 이름>.pem ec2-user@<인스턴스 퍼블릭 IPv4>
```

![Untitled](/assets/img/Infra/AWS/ec2/5.png)

접속되면 Apache `httpd`를 설치해 웹 서버로 동작시킨다.

```bash
sudo yum install -y httpd
```

## 5. Cloud9 환경 생성

> **Cloud9이란?** 브라우저에서 코드를 작성·실행하는 **클라우드 IDE**. 생성하면 내부적으로 EC2 인스턴스를 하나 만들어 그 위에서 동작한다.

Cloud9에서 **환경 생성**을 누르고, 새 EC2 인스턴스(프리 티어)와 절전 시간 제한을 설정한다. 연결은 SSH, VPC는 공부용으로 만든 VPC를 선택한다.

## 📝 정리

```
EC2 & Cloud9
├─ 클라우드  종량제 · 빠른 구축 · 고가용성
├─ 계층      리전 → 가용 영역(AZ)
├─ EC2       AMI 선택 → 키페어 → 보안그룹 → SSH 접속
└─ Cloud9    브라우저 IDE(내부적으로 EC2 생성)
```

| 개념 | 한 줄 정의 |
|------|------|
| **EC2** | 클라우드 가상 서버 |
| **AMI** | 인스턴스 생성용 이미지 |
| **가용 영역** | 리전 내 분리된 데이터센터 |

EC2는 AWS의 출발점이 되는 **가상 서버**다. 핵심 흐름은 **AMI 선택 → 키페어 생성 → 보안 그룹 설정 → SSH 접속**이며, 이 위에 웹 서버를 올리면 곧바로 서비스를 시작할 수 있다.
