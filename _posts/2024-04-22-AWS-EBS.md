## 📌 들어가며

이번 글에서는 AWS의 **EBS(Elastic Block Store)**를 정리한다. EBS는 EC2에 붙이는 **네트워크 기반 블록 스토리지**로, 인스턴스를 종료해도 데이터가 유지되는 **가상 하드디스크**라고 보면 된다. 볼륨을 생성해 EC2에 연결하고, 리눅스에서 마운트해 사용하는 전체 흐름을 실습한다.

> **EBS란?** EC2에 연결해 쓰는 **블록 수준 스토리지 볼륨**. 인스턴스와 독립적으로 존재하므로 인스턴스를 종료해도 데이터가 남고, 다른 인스턴스에 다시 붙일 수도 있다. **하나의 볼륨은 같은 가용 영역(AZ)의 인스턴스에만** 연결된다.

---

## 1. EBS 사용 흐름

볼륨은 **생성 → 연결 → 포맷 → 마운트**의 순서로 실제 디스크처럼 쓸 수 있게 된다.

```
① 볼륨 생성      (EC2 - 볼륨 탭, AZ 일치 필수)
② 인스턴스 연결   (작업 - 볼륨 연결, /dev/sdb)
③ 파일시스템 생성 (mkfs)
④ 마운트         (mount → 디렉터리에 연결)
```

---

## 2. 볼륨 생성

`EC2 → 볼륨` 탭에서 **볼륨 생성**을 누른다. 예시에서는 `web01` 인스턴스에 붙일 gp2 타입 1GB 볼륨을 만들고, 이름을 `web01-add`로 지정한다.

> ⚠️ **가용 영역(AZ)이 반드시 일치**해야 한다. 볼륨은 네트워크로 붙지만 **같은 AZ 안에서만** 연결 가능하므로, 대상 인스턴스(`web01-root`)와 **동일한 AZ**에 볼륨을 만들어야 연결된다.

## 3. 인스턴스에 볼륨 연결

만든 볼륨을 선택하고 **작업 → 볼륨 연결**을 눌러, 대상 인스턴스를 고르고 디바이스 이름(`/dev/sdb`)으로 연결한다.

## 4. 파일시스템 생성 & 마운트

연결만으로는 쓸 수 없고, **포맷(mkfs) 후 마운트**해야 실제 경로에서 사용할 수 있다.

```bash
lsblk                          # 연결된 블록 디바이스 확인

sudo mkfs -t xfs /dev/xvdb     # 파일시스템 생성(포맷)
sudo mount /dev/xvdb /mnt      # /mnt 에 마운트
sudo umount /mnt               # 언마운트
```

| 명령 | 역할 |
|------|------|
| `lsblk` | 블록 디바이스 목록(연결만 되면 조회됨) |
| `mkfs -t xfs` | 볼륨을 XFS로 포맷 |
| `mount` / `umount` | 디렉터리에 연결 / 해제 |
| `df -h` | **마운트된** 파일시스템 용량 조회 |

> 💡 **`lsblk`와 `df -h`의 차이** — `lsblk`는 **마운트와 무관하게** 볼륨이 연결되면 바로 조회되지만, `df -h`는 **마운트가 되어야만** 조회된다. 즉 "연결됐는데 `df`에 안 보인다"면 마운트를 안 한 것이다.

## 📝 정리

```
EBS(Elastic Block Store)
├─ 개념   EC2에 붙이는 네트워크 블록 스토리지
├─ 제약   같은 AZ 안에서만 연결 가능
├─ 흐름   생성 → 연결 → mkfs → mount
└─ 조회   lsblk(연결) vs df -h(마운트 후)
```

| 개념 | 한 줄 정의 |
|------|------|
| **EBS** | 인스턴스와 독립된 블록 볼륨 |
| **AZ 일치** | 볼륨·인스턴스가 같은 가용 영역이어야 연결 |
| **mkfs → mount** | 포맷 후 디렉터리에 연결해야 사용 |

EBS는 EC2의 **탈부착 가능한 하드디스크**다. 핵심은 **같은 AZ에 만들어 연결하고, 포맷·마운트를 거쳐야 쓸 수 있다**는 것. 인스턴스와 수명이 분리되어 있어 데이터 보존과 이전에 유리하다.
