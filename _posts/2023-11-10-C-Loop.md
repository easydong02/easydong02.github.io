## 📌 들어가며

같은 작업을 여러 번 반복해야 할 때, 그걸 손으로 일일이 쓰는 건 프로그래머의 방식이 아니다. 반복을 코드에게 맡기는 것이 **순환문(loop)**이다. 이번 글에서는 `for`, `while`, `do ~ while`과 중첩 순환문, 그리고 `break` / `continue`를 정리한다.

| 순환문 | 특징 |
|------|------|
| `for` | 반복 횟수가 명확할 때 |
| `while` | 조건이 참인 동안 반복 |
| `do ~ while` | **최소 한 번은 실행** (현대 언어에선 잘 안 쓰임) |

---

## 1. for 순환문

![Desktop View](/assets/img/Programming-Language/C/Loop/1.png){: width="100%" }

```c
for (①초기식; ②조건식; ④증감식) {
    ③수행문;
}
```

실행 순서가 번호와 다르다는 점이 핵심이다.

```
① 초기식    최초에 딱 한 번만 수행
   ↓
② 조건식 ── 거짓 ──► for문 종료
   │ 참
   ▼
③ 수행문    조건이 참이면 실행
   ↓
④ 증감식    수행문이 끝나면 실행 → 다시 ②로
```

> 💡 **순환문을 작성할 때 반드시 확인할 3가지**
> 1. 몇 번 순환하는가?
> 2. 인덱스의 시작값과 끝값은?
> 3. **순환이 끝난 뒤 인덱스 값은?** ← 흔히 빼먹지만, for문 이후 문장을 정확히 계산하려면 꼭 고려해야 한다.

**작성 TIP:**

```c
// ① n번 반복 (횟수가 핵심)
for (int i = 0; i < n; i++) { ... }

// ② a ~ b 반복 (시작·끝값이 핵심)
for (int i = a; i <= b; i++) { ... }
```

> ⚠️ `for`문 안에서 선언한 `int i`는 **지역변수**라 for문 바깥에서 못 쓴다. for문 이후에도 쓰려면 for문 시작 전에 선언한다.

![Desktop View](/assets/img/Programming-Language/C/Loop/2.png){: width="100%" }

### 예제 ① — 구구단 2단

```c
for (int i = 1; i <= 9; i++)
    printf("2 * %d = %d\n", i, 2 * i);
```

![Desktop View](/assets/img/Programming-Language/C/Loop/3.png){: width="100%" }

`i`는 1부터 9까지, 총 9번 실행된다. 한 번 돌 때마다 `i`가 1씩 커지므로 `2*i`가 2의 배수로 출력된다.

![Desktop View](/assets/img/Programming-Language/C/Loop/4.png){: width="100%" }

### 예제 ② — 1~n 누적합

```c
int n = 100, sum = 0;
for (int i = 1; i <= n; i++)
    sum += i;          // sum = sum + i
printf("합 = %d\n", sum);   // for문 '밖'에서 한 번만 출력
```

![Desktop View](/assets/img/Programming-Language/C/Loop/5.png){: width="100%" }

`+=`는 누적 연산자다. `i`가 1이면 `sum`(0)에 1을 더해 저장하고, 2면 누적값 1에 2를 더하고… 100까지 진행한다. **`printf`를 for문 밖에 둬서 총합을 한 번만 출력**하는 게 포인트다.

![Desktop View](/assets/img/Programming-Language/C/Loop/6.png){: width="100%" }

## 2. while 순환문

`while`은 **조건식이 참인 동안** 블록을 반복한다.

```c
while (조건식) { ... }
```

> 💡 **`while`과 `for`는 100% 호환**된다. for로 만든 건 while로, while로 만든 건 for로 바꿀 수 있다.

앞의 구구단을 while로 바꿔보자.

![Desktop View](/assets/img/Programming-Language/C/Loop/7.png){: width="100%" }

`for`와 달리 `while`의 괄호에는 **조건만** 들어간다. 그래서 `i`의 선언과 증감식(`i++`)은 **따로** 적어야 한다.

![Desktop View](/assets/img/Programming-Language/C/Loop/8.png){: width="100%" }

## 3. 중첩 순환문 (nested loop)

**순환문 안에 있는 순환문**이다. 순환문 하나로는 구구단 한 단밖에 못 만들지만, 중첩하면 **전체 구구단**을 만들 수 있다.

![Desktop View](/assets/img/Programming-Language/C/Loop/9.png){: width="100%" }

```
바깥 for (i: 2~9)   ← 단(段)
   안쪽 for (j: 1~9)   ← 곱하는 수
       printf(i * j)

i=2 고정 → j 1~9 전부 실행 (2단 완성)
i=3 고정 → j 1~9 전부 실행 (3단 완성)
...
```

바깥 for가 한 번 도는 동안 안쪽 for는 **처음부터 끝까지** 전부 돈다. 그래서 `i`가 2로 고정된 채 `j`만 1~9를 돌아 2단이 완성되고, 그다음 `i`가 3이 되어 3단… 9단까지 만들어진다.

![Desktop View](/assets/img/Programming-Language/C/Loop/10.png){: width="100%" }

## 4. Break와 Continue

| 키워드 | 동작 |
|------|------|
| **`break`** | 자기를 감싸는 **가장 가까운 순환문을 종료** |
| **`continue`** | 자기를 감싸는 **가장 가까운 순환문의 처음으로** 돌아감 |

> ⚠️ 핵심은 **'순환문'을 대상으로 한다**는 것이다. `if` 안에 있어도 `break`는 `if`가 아니라 **순환문**을 빠져나간다. (처음엔 "왜 for문을 빠져나가지?"하고 헷갈리기 쉽다!)

### break 예제

![Desktop View](/assets/img/Programming-Language/C/Loop/11.png){: width="100%" }

`for`의 조건식 자리를 비우면 항상 참이라 **무한 반복**된다. 하지만 `if` 안에서 `i`가 7의 배수가 될 때 `break`로 for문을 종료한다. `printf`가 `if`보다 뒤에 있으므로 `i`는 6까지만 출력된다.

![Desktop View](/assets/img/Programming-Language/C/Loop/12.png){: width="100%" }

### continue 예제

![Desktop View](/assets/img/Programming-Language/C/Loop/13.png){: width="100%" }

`i`가 2의 배수일 때 `continue`를 만나면 **`printf`로 가지 않고 순환문 처음으로** 돌아간다. 따라서 0~10 중 **짝수는 건너뛰고 홀수만** 출력된다.

## 5. do ~ while 순환문

요즘엔 잘 쓰이지 않지만, `while`과의 결정적 차이가 하나 있다.

![Desktop View](/assets/img/Programming-Language/C/Loop/15.png){: width="100%" }

`i = 1`, 조건이 둘 다 `i < 0`인 경우를 보자.

| 순환문 | 조건이 처음부터 거짓이면 |
|------|------|
| `while` | **한 번도 실행 안 됨** (조건 먼저 검사) |
| `do ~ while` | **일단 한 번은 실행됨** (실행 후 조건 검사) |

`do ~ while`은 일단 `do!` 하고 나서 조건을 검사하기 때문에, **조건이 맞지 않아도 최소 한 번은 실행**된다.

![Desktop View](/assets/img/Programming-Language/C/Loop/16.png){: width="100%" }

보이는 것처럼 `do ~ while`의 `printf`만 실행됐다.

## 📝 정리

```
순환문(loop)
├─ for        for(초기식; 조건식; 증감식) { 수행문 }  실행순서 ①②③④②③④…
├─ while      조건만 괄호에, 선언·증감은 따로 (for와 100% 호환)
├─ do~while   조건이 거짓이어도 최소 1회 실행
├─ 중첩       순환문 안의 순환문 → 구구단 전체 등
└─ 제어       break(순환문 종료) · continue(순환문 처음으로)
```

| 개념 | 한 줄 정의 |
|------|------|
| **for** | 반복 횟수가 명확할 때 쓰는 순환문 |
| **while** | 조건이 참인 동안 반복 |
| **do~while** | 최소 한 번은 실행되는 순환문 |
| **break** | 가장 가까운 순환문을 종료 |
| **continue** | 가장 가까운 순환문의 처음으로 |

순환문은 반복을 코드에 위임하는 도구다. 특히 `break`/`continue`가 **조건문이 아니라 순환문**을 대상으로 한다는 점, 그리고 for문 종료 후의 인덱스 값까지 고려하는 습관을 들이면 실수를 크게 줄일 수 있다.
